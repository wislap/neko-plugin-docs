状态持久化模式配置（默认 "off"，需在 toml 中启用）。 - "auto": 所有 entry 执行后自动保存状态 - "manual": 只在 freeze 时保存，或在 @plugin_entry(persist=True) 显式启用 - "off": 完全禁用持久化功能（默认）

优先级：toml [plugin_state].persist_mode > 类属性 > 默认值

Type:

include_router(router, *, prefix='')[源代码]¶

注册一个 PluginRouter（支持动态加载）

将 Router 中定义的所有入口点注册到主插件中。 Router 中的入口点可以访问主插件的 ctx、config、plugins 等功能。

参数:

  • router (PluginRouter) -- PluginRouter 实例

  • prefix (str) -- 可选的前缀，会覆盖 Router 自身的 prefix

返回类型:

None

示例

>>> self.include_router(DebugRouter())
>>> self.include_router(MemoryRouter(), prefix="mem_")

卸载一个 PluginRouter（支持动态卸载）

参数:

router (PluginRouter | str) -- PluginRouter 实例或 Router 名称字符串

返回:

True 如果成功卸载，False 如果未找到

返回类型:

bool

示例

>>> self.exclude_router(my_router)  # 通过实例卸载
>>> self.exclude_router("MyRouter")  # 通过名称卸载

True 如果注册成功

bool

>>> async def on_startup(self):

async register_dynamic_entry(entry_id, handler, name='', description='', input_schema=None, kind='action', auto_start=False)[源代码]¶

动态注册一个 entry

在运行时创建并注册一个新的 entry，会通知主进程更新 entry 列表。

参数:

  • entry_id (str) -- entry 的唯一标识符

  • handler (Callable) -- 处理函数（async 或 sync）

  • name (str) -- 显示名称（默认使用 entry_id）

  • description (str) -- 描述信息

  • input_schema (Dict[str, Any] | None) -- 输入参数的 JSON Schema

  • kind (str) -- entry 类型（action/service/hook/custom）

  • auto_start (bool) -- 是否自动启动

返回:

True 如果注册成功

返回类型:

bool

示例

>>> async def my_handler(self, arg1: str, **_):
...     return ok(data={"result": arg1})
>>> await self.register_dynamic_entry("my_entry", my_handler, name="My Entry")

enable_file_logging(log_level=None, max_bytes=None, backup_count=None, max_files=None)[源代码]¶

启用插件文件日志功能（使用loguru）

为插件创建独立的文件日志，日志文件保存在插件的logs目录下。 日志会同时输出到文件和控制台（终端）。 自动管理日志文件数量，支持日志轮转。

参数:

  • log_level (str | None) -- 日志级别（字符串："DEBUG", "INFO", "WARNING", "ERROR"），默认使用配置中的PLUGIN_LOG_LEVEL

  • max_bytes (int | None) -- 单个日志文件最大大小（字节），默认使用配置中的PLUGIN_LOG_MAX_BYTES

  • backup_count (int | None) -- 保留的备份文件数量，默认使用配置中的PLUGIN_LOG_BACKUP_COUNT

  • max_files (int | None) -- 最多保留的日志文件总数，默认使用配置中的PLUGIN_LOG_MAX_FILES

返回:

配置好的loguru logger实例（已添加文件handler和控制台handler）

返回类型:

Any

使用示例:

```python class MyPlugin(NekoPluginBase):

def __init__(self, ctx):

super().__init__(ctx) # 启用文件日志（同时输出到文件和控制台） self.file_logger = self.enable_file_logging(log_level="DEBUG") # 使用file_logger记录日志，会同时显示在终端和保存到文件 self.file_logger.info("Plugin initialized")

```

注意:

  • 日志文件保存在 {plugin_dir}/logs/ 目录下

  • 日志文件名格式：`{plugin_id}_{YYYYMMDD_HHMMSS}.log`（包含日期和时间）

  • 日志会同时输出到文件和控制台（终端）

  • 当日志文件达到最大大小时会自动轮转

  • 超过最大文件数量限制的旧日志会自动删除
