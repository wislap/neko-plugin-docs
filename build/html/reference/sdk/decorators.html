plugin.sdk.decorators.on_event(*, event_type, id=None, name=None, description='', input_schema=None, kind='action', auto_start=False, persist=None, checkpoint=None, metadata=None, extra=None)[源代码]¶

通用事件装饰器。 - event_type: "plugin_entry" / "lifecycle" / "message" / "timer" ... - id: 在"本插件内部"的事件 id（不带插件 id）。

None 时从被装饰函数名自动推断。

  • persist: 执行后是否保存状态（None=遵循 __persist_mode__）

  • checkpoint: persist 的向后兼容别名

  • metadata: 额外的元数据字典

  • extra: metadata 的向后兼容别名（已弃用）

参数:

  • event_type (str)

  • id (str | None)

  • name (str | None)

  • description (str)

  • input_schema (dict | None)

  • kind (Literal['service', 'action', 'hook', 'custom', 'lifecycle', 'consumer', 'timer'])

  • auto_start (bool)

  • persist (bool | None)

  • checkpoint (bool | None)

  • metadata (dict | None)

  • extra (dict | None)

plugin.sdk.decorators.worker(timeout=30.0, priority=0)[源代码]¶

Worker 模式装饰器

标记函数应该在 worker 线程池中执行，而不是在命令循环线程中执行。 可以叠加在 @plugin_entry、@lifecycle、@bus_subscribe 等装饰器上。

参数:

  • timeout (float) -- 超时时间（秒），默认 30.0

  • priority (int) -- 优先级（数字越大优先级越高），默认 0

示例

@worker(timeout=30.0) @plugin_entry(id="sync_task") def sync_task(self, param: str):

# 同步代码，会在 worker 线程池中执行 return ok(data={"result": param})

plugin.sdk.decorators.plugin_entry(id=None, name=None, description='', input_schema=None, params=None, kind='action', auto_start=False, persist=None, checkpoint=None, timeout=None, metadata=None, extra=None)[源代码]¶

语法糖：专门用来声明"对外可调用入口"的装饰器。 本质上是 on_event(event_type="plugin_entry").

参数:

  • id (str | None) -- Entry ID. None → auto-inferred from function name.

  • name (str | None) -- Display name. None → same as id.

  • input_schema (dict | None) -- Explicit JSON Schema dict. None → auto-inferred from function signature (or params model if provided).

  • params (type | None) --

    Optional Pydantic BaseModel subclass. When provided: - input_schema is auto-extracted via model.model_json_schema(). - The model class is attached to the function for optional runtime

    validation (accessible via _PARAMS_MODEL_ATTR).

  • persist (bool | None) -- 执行后是否保存状态 - None: 遵循类级别 __persist_mode__ 配置 - True: 强制启用状态保存 - False: 强制禁用状态保存

  • checkpoint (bool | None) -- persist 的向后兼容别名

  • timeout (float | None) -- 自定义超时时间（秒） - None: 使用默认超时（PLUGIN_TRIGGER_TIMEOUT，默认 10 秒） - 0 或负数: 禁用超时检测（无限等待） - 正数: 使用指定的超时时间

  • metadata (dict | None) -- 额外的元数据字典

  • extra (dict | None) -- metadata 的向后兼容别名（已弃用）

  • description (str)

  • kind (Literal['service', 'action', 'hook', 'custom', 'lifecycle', 'consumer', 'timer'])

  • auto_start (bool)

返回类型:

Callable

plugin.sdk.decorators.lifecycle(*, id, name=None, description='', metadata=None, extra=None)[源代码]¶

生命周期事件装饰器

支持的生命周期事件： - startup: 插件启动时调用 - shutdown: 插件停止时调用 - reload: 插件重载时调用 - freeze: 插件冻结前调用（可用于清理资源、保存额外状态） - unfreeze: 插件从冻结状态恢复后调用（可用于重新初始化资源） - config_change: 配置热更新时调用（接收 old_config, new_config, mode 参数）

参数:

  • id (Literal['startup', 'shutdown', 'reload', 'freeze', 'unfreeze', 'config_change'])

  • name (str | None)

  • description (str)

  • metadata (dict | None)

  • extra (dict | None)

返回类型:

Callable

plugin.sdk.decorators.custom_event(*, event_type, id, name=None, description='', input_schema=None, kind='custom', auto_start=False, trigger_method='message', metadata=None, extra=None)[源代码]¶

自定义事件装饰器：允许插件定义全新的事件类型。

参数:

  • event_type (str) -- 自定义事件类型名称（例如 "file_change", "user_action" 等）

  • id (str) -- 事件ID（在插件内部唯一）

  • name (str | None) -- 显示名称

  • description (str) -- 事件描述

  • input_schema (dict | None) -- 输入参数schema（JSON Schema格式）

  • kind (Literal['service', 'action', 'hook', 'custom', 'lifecycle', 'consumer', 'timer']) -- 事件种类，默认为 "custom"

  • auto_start (bool) -- 是否自动启动（如果为True，会在插件启动时自动执行）

  • trigger_method (str) -- 触发方式 - "message": 通过消息队列触发（推荐，异步） - "command": 通过命令队列触发（同步，类似 plugin_entry） - "auto": 自动启动（类似 timer auto_start）

  • metadata (dict | None) -- 额外配置信息

  • extra (dict | None) -- metadata 的向后兼容别名（已弃用）

返回:

装饰器函数

返回类型:

Callable

示例

@custom_event(

event_type="file_change", id="on_file_modified", name="文件修改事件", description="当文件被修改时触发", trigger_method="message"

) def handle_file_change(self, file_path: str, action: str):

self.logger.info(f"File {file_path} was {action}")

plugin.sdk.decorators.hook(target: str, timing: Literal['before', 'after', 'around', 'replace'] = 'before', priority: int = 0, condition: str | None = None) → Callable[[_HookFn], _HookFn]

Hook 装饰器

用于声明一个方法为 Hook，可以在目标 entry 执行前/后/周围执行。 支持插件内部中间件和跨插件 Hook 两种场景。

参数:

  • target (str) -- Hook 目标 - 插件内: "entry_id" - Hook 当前插件/Router 的指定 entry - 插件内: "*" - Hook 当前插件/Router 的所有 entry - 跨插件: "plugin_id.entry_id" - Hook 其他插件的指定 entry

  • timing (Literal['before', 'after', 'around', 'replace']) --

    执行时机 - "before": 在目标 entry 执行前执行

    • 返回 None: 继续执行

    • 返回 dict: 阻止执行，直接返回该结果

    • 返回修改后的 params: 修改参数后继续执行

    • "after": 在目标 entry 执行后执行

      • 接收 result 参数，可以修改返回值

    • "around": 包裹目标 entry

      • 接收 next_handler 参数，可以控制是否执行原始 handler

    • "replace": 完全替换目标 entry

      • 接收 original_handler 参数，可以选择性调用

  • priority (int) -- 优先级（越大越先执行），默认 0

  • condition (str | None) -- 条件表达式或方法名（可选） - 字符串: 当前类的方法名，返回 True 才执行 Hook

返回:

装饰器函数

返回类型:

Callable[[_HookFn], _HookFn]

Example - 插件内中间件:
>>> class MyRouter(PluginRouter):
...     @hook(target="*", timing="before")
...     async def log_all_calls(self, entry_id: str, params: dict, **_):
...         '''Hook 所有 entry，记录调用日志'''
...         self.logger.info(f"Calling {entry_id} with {params}")
...         return None  # 继续执行
...
...     @hook(target="save", timing="before", priority=10)
...     async def validate_save(self, params: dict, **_):
...         '''验证 save entry 的参数'''
...         if not params.get("data", {}).get("name"):
...             return fail(message="name is required")  # 阻止执行
...         return None
...
...     @hook(target="load", timing="after")
...     async def cache_result(self, entry_id: str, result: dict, **_):
...         '''缓存 load 结果'''
...         self._cache[entry_id] = result
...         return result  # 可以修改返回值
Example - 跨插件 Hook (扩展插件):
>>> class ExtensionRouter(PluginRouter):
...     @hook(target="core_plugin.save", timing="before")
...     async def extend_save(self, params: dict, **_):
...         '''Hook 其他插件的 save entry'''
...         params["extended"] = True
...         return params  # 修改参数后继续
...
...     @hook(target="core_plugin.save", timing="around")
...     async def wrap_save(self, params: dict, next_handler, **_):
...         '''包裹 save entry'''
...         self.logger.info("Before save")
...         result = await next_handler(params)
...         self.logger.info("After save")
...         return result

备注

  • Hook 方法的签名根据 timing 不同而不同:

    • before: (self, entry_id: str, params: dict, **_) -> Optional[dict]

    • after: (self, entry_id: str, params: dict, result: dict, **_) -> dict

    • around: (self, entry_id: str, params: dict, next_handler: Callable, **_) -> dict

    • replace: (self, entry_id: str, params: dict, original_handler: Callable, **_) -> dict

  • 跨插件 Hook 需要在 plugin.toml 中声明权限
