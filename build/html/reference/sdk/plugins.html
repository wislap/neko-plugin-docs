Plugins (跨插件调用)¶

exception plugin.sdk.plugins.PluginCallError[源代码]¶

基类：RuntimeError

插件调用错误

当插件间通信失败时抛出此异常,例如: - 目标插件不存在 - 事件ID无效 - 调用超时 - 参数格式错误

class plugin.sdk.plugins.Plugins[源代码]¶

基类：object

插件管理器

提供插件间通信和查询功能。通过self.plugins访问此类的实例。

list(filters=None, *, timeout=5.0)[源代码]¶

查询插件列表

获取系统中所有插件的信息,可以通过filters参数过滤。

参数:

  • filters (Dict[str, Any] | None) -- 过滤条件字典,例如 {"status": "running", "include_events": True}

  • timeout (float) -- 超时时间(秒)

返回:

{
"plugins": [

{"plugin_id": "...", "name": "...", "status": "...", ...}, ...

]

}

返回类型:

包含插件列表的字典,格式为

抛出:

  • PluginCallError -- 如果ctx.query_plugins不可用

  • TimeoutError -- 如果查询超时

示例

>>> plugins = self.plugins.list()
>>> running_plugins = self.plugins.list(filters={"status": "running"})
call_entry(ref, params, *, timeout=10.0)[源代码]¶

调用其他插件的entry

通过"plugin_id:entry_id"格式的引用调用其他插件的plugin_entry。

参数:

  • ref (str) -- 插件entry引用,格式为"plugin_id:entry_id"

  • params (Dict[str, Any]) -- 传递给entry的参数字典

  • timeout (float) -- 超时时间(秒)

返回:

目标entry的返回结果

抛出:

  • PluginCallError -- 如果引用格式无效或调用失败

  • TimeoutError -- 如果调用超时

返回类型:

Any

示例

>>> result = self.plugins.call_entry(
...     "data_processor:process",
...     {"data": [1, 2, 3]}
... )
call_event(ref, params, *, timeout=10.0)[源代码]¶

调用其他插件的自定义事件

通过"plugin_id:event_type:event_id"格式的引用调用其他插件的自定义事件。

参数:

  • ref (str) -- 事件引用,格式为"plugin_id:event_type:event_id"

  • params (Dict[str, Any]) -- 传递给事件处理器的参数字典

  • timeout (float) -- 超时时间(秒)

返回:

事件处理器的返回结果

抛出:

  • PluginCallError -- 如果引用格式无效或调用失败

  • TimeoutError -- 如果调用超时

返回类型:

Any

示例

>>> result = self.plugins.call_event(
...     "notifier:custom_event:send_notification",
...     {"message": "Hello"}
... )
call(*, plugin_id, event_type, event_id, params, timeout=10.0)[源代码]¶

通用插件事件调用方法

底层方法,用于调用其他插件的任意事件。通常建议使用call_entry()或call_event()。

参数:

  • plugin_id (str) -- 目标插件ID

  • event_type (str) -- 事件类型("plugin_entry", "lifecycle", "message", "timer"等)

  • event_id (str) -- 事件ID

  • params (Dict[str, Any]) -- 参数字典

  • timeout (float) -- 超时时间(秒)

返回:

事件处理器的返回结果

抛出:

  • PluginCallError -- 如果ctx.trigger_plugin_event不可用

  • RuntimeError -- 如果调用失败

  • TimeoutError -- 如果调用超时

返回类型:

Any

示例

>>> result = self.plugins.call(
...     plugin_id="other_plugin",
...     event_type="plugin_entry",
...     event_id="action",
...     params={"param": "value"}
... )
require(plugin_id, *, timeout=5.0)[源代码]¶

检查必需的插件是否存在

验证指定的插件是否已加载。如果插件不存在,抛出PluginCallError异常。 用于声明插件依赖关系。

参数:

  • plugin_id (str) -- 必需的插件ID

  • timeout (float) -- 超时时间(秒)

抛出:

  • PluginCallError -- 如果必需的插件不存在

  • TimeoutError -- 如果查询超时

返回类型:

None

示例

>>> def __init__(self, ctx):
...     super().__init__(ctx)
...     # 声明依赖
...     self.plugins.require("data_processor")
...     self.plugins.require("notifier")
